Differential Gene Expression

Microarray analysis and RNA-seq generate large amounts of data, which require statistical analysis  to identify differentially expressed genes between two or more biological samples. This involves comparing the expression levels of each gene and calculating a statistical significance value, such as a p-value or a false discovery rate(FDR). Genes with a significant difference in expression levels are considered to be differentially expressed.

There are common methods and software packages used in differential gene expression analysis.

  1. t-test: t-test is a statistical test used to determine if the means of two samples are significantly different from each other. It is a simple and commonly used method for differential gene expression analysis.
  1. Analysis of variance (ANOVA): ANOVA is used to test for differences between two or more groups. ANOVA is commonly used in differential expression analysis to test for gene expression differences among multiple groups.
  1. DESeq2: DESeq2 is a widely used software package for differential gene expression analysis in RNA-seq data. It models count data using a negative binomial distribution and estimates the dispersion of the data using maximum likelihood.[3]
  1. edgeR: edgeR is another widely used software package for differential gene expression analysis in RNA-seq data. It models count data using a negative binomial distribution and estimates the dispersion of the data using empirical Bayes methods.[4]
  1. Limma: Limma is a software package used for differential gene expression analysis in microarray and RNA-seq data. It uses a linear modeling approach to estimate the effect of different experimental conditions on gene expression. Limma also performs normalization of the data to correct for differences in array quality and other technical effects.[5]

In this assigment, I choose DESeq2 to complete Differential Gene Expression Analysis. The following are the steps typically involved in using DESeq2 for differential gene expression analysis[4]:

  1. Normalization: The count matrix is normalized to correct for differences in sequencing depth and other technical effects. DESeq2 uses a normalization method based on the median of ratios between each sample and a geometric mean of all samples.
  1. Differential expression analysis: DESeq2 uses a negative binomial distribution to model the count data and estimate the dispersion of the data using maximum likelihood estimation. Differential expression analysis is then performed using a generalized linear model, which includes terms for the experimental design and sample covariates. DESeq2 calculates p-values and log2 fold changes for each gene, which are used to identify differentially expressed genes.
  1. Multiple testing correction: The p-values are adjusted for multiple testing using a method such as the Benjamini-Hochberg procedure to control the false discovery rate.

Thresholded over-representation analysis

Thresholded over-representation analysis (ORA) is a statistical method used for identifying enriched gene sets in biological datasets. The following are the steps involved in thresholded ORA[6, 7]:

  1. Gene set definition: A gene set is defined as a group of genes that share a common function or are involved in a specific biological pathway. The gene set can be defined based on prior knowledge from databases such as Gene Ontology (GO) or Kyoto Encyclopedia of Genes and Genomes (KEGG).
  1. Thresholding: The thresholding step involves filtering the genes in the dataset based on a predefined threshold, such as a fold change or p-value. This reduces the number of genes analyzed, which improves the statistical power and reduces false positives.
  1. Over-representation analysis: Over-representation analysis involves comparing the number of genes in the gene set to the number of genes in the dataset that pass the threshold. A statistical test, such as Fisher's exact test or chi-square test, is used to determine if the gene set is over-represented in the dataset compared to what would be expected by chance.
  1. Multiple testing correction: Since multiple gene sets are typically tested, multiple testing correction is applied to control the false discovery rate. Common methods for multiple testing correction include the Benjamini-Hochberg procedure or the Bonferroni correction.
  1. Functional analysis: The enriched gene sets can be further analyzed for functional enrichment using tools such as Enrichr, DAVID, or GSEA. This involves identifying enriched biological pathways and functions that are associated with the differentially expressed genes.
